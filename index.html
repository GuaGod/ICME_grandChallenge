Grand Challenge on Dynamic Gaussian Splatting Compression

Challenge Overview
Dynamic 3D Gaussian Splatting (DyGS) enables real-time, high-fidelity rendering of dynamic scenes and is increasingly used in AR/VR/XR and volumetric video. However, naively storing a time-varying set of Gaussian primitives leads to prohibitive storage and bandwidth costs. While many compression approaches have been proposed, progress is slowed by the lack of a standardized benchmark that evaluates rate, rendering quality, and online decoding cost under a unified protocol.
DyGS-Bench focuses on a realistic deployment setting where only the reconstructed native Gaussian sequence is available (no original multi-view training images/videos). The goal of this Grand Challenge is to advance practical codecs that compress dynamic Gaussian assets into a compact bitstream and reconstruct renderable Gaussians efficiently.

Task
Given a native dynamic Gaussian sequence {G_t} and a fixed camera set C, design an encoder (offline) and decoder (online, measured) that:
  • Produces a compact bitstream from {G_t} (offline encoding is not time-constrained).
  • Determine code stream size for coded Gaussian.
  • Achieves high rendering quality when rendering and temporal consistency Ghat_t under C compared with renderings of {G_t}.
Datasets
DyGS-Bench focuses on a native Gaussians-only setting: the input is a reconstructed dynamic 3D Gaussian sequence (no original multi-view training RGB/video data). Each sequence includes:
  • Native Gaussian assets: time-varying Gaussians (per-frame PLY).
  • Camera set: calibrated camera parameters for rendering-based evaluation.
Baseline
LightGaussian for Dynamic Gaussian Compression Challenge.
Overview of the Baseline Framework, LightGaussian, for the Dynamic Gaussian Compression Challenge. The provided baseline pipeline for participants, incorporating Gaussian pruning, Spherical Harmonics (SH) distillation, and Vector Quantization.
